# CAP 정리

> 분산 시스템을 설계할때는 CAP에 대해 이해가 선행되어야 한다

- CAP는 Consistency(일관성), Availability(가용성), Partition Tolerance theorem(파티션 감내)로 구성된다
  - Consistency: 분산 시스템에 접속하는 모든 클라이언트는 어떤 노드에 접속했느냐에 관계 없이 언제나 같은 데이터를 보게 되어야 한다
  - Availability: 분산 시스템에 접속하는 클라이언트는 일부 노드에 장애가 발생하더라도 항상 응답을 받을 수 있어야 한다.
  - Partition Tolerance Theorm: 파티션은 두 노드 사이에 장애가 발생하였음을 의미하며, 네트워크에 장애 파티션이 생기더라도 시스템은 계속 동작해야 한다는 것을 의미한다
- CAP 정리는 이들 사이에 두가지를 충족하려면 나머지 하나는 반드시 희생되어야 한다는 것을 의미한다
- P는 분산 시스템에서 반드시 지켜져야 하므로, CP/AP가 된다
  - 분산 시스템에서 네트워크 장애는 피할 수 없다

## CP

- 가용성을 희생한 방식이다
- 노드 하나가 네트워크 문제가 생겼을 경우, 쓰기 불가로 바뀌어 일관성을 지킨다
  - Timeout, Error로 내보낸다
- 복구가 되면 전체 노드를 sync시킨 후, 활동을 재개한다
- 금융권/분산락의 경우 이 방식을 사용한다

## AP

- 일관성을 희생한 방식이다
- 노드하나가 네트워크 문제가 생겼을 경우, 해당 노드를 사용하지 않는다
- 최신 상태로 sync를 맞추고, 활동을 계속하여 장애가 없이 실행 되도록한다
- 좋아요 댓글 시스템처럼 데이터에 민감하지 않은 경우 해당 방식을 사용한다
