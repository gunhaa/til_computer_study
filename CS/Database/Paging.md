# Paging과 Sort

## 발생한 문제

- user admin에서 키워드로 검색을 하여도 유저가 검색이 안되는 경우가 발생하였다
  - '건하'로 특정하면 검색이 되지만, '건'으로 검색하면 검색 결과는 30건이지만 원하는 '황건하' 유저가 검색이 되지 않았다
- be에서는 mysql limit,offset을 통해 페이징 처리를 진행하고 있었다
- 원인을 찾기 위해 데이터를 비교해보니, limit/offset을 제거하고 검색하면 원하는 데이터가 검색이 되었다
- limit, offset을 제거한 결과와 페이징처리 된 결과를 비교해보니, 중복된 결과가 1,2페이지에서 여러개가 발견되었다
- 쿼리를 확인해보니 order by를 create_date로 진행하고 있었는데, 일부 user들이 create_date가 같은 상태였다
  - 조건이 같을 경우, db engine은 빠르게 처리되는 결과를 반환하기에 결과의 순서를 보장하지 않아서 결과가 바뀔 수 있어, 명확한 기준이 필요하다
  - 기준을 명확하게 세우기 위한 추가 조건이 필요하다(order by create_date asc, id asc)
- 이 현상은 인덱스 유무나 db engine의 실행 계획에 따라 운영 환경에서만 간헐적으로 발생할 수 있어 발견이 까다로운 문제였다
